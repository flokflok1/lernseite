# AP1 FISI Pruefungsanalyse Baden-Wuerttemberg

## Uebersicht

**Zielgruppe:** Fachinformatiker Systemintegration (FISI)
**Pruefung:** Abschlusspruefung Teil 1 (AP1)
**Bundesland:** Baden-Wuerttemberg
**Bearbeitungszeit:** 90 Minuten
**Analysierte Pruefungen:** 2017-2024 (Sommer & Winter)

---

## 1. Pruefungsstruktur

### Neue Pruefungsordnung (ab 2020)
Die AP1 ist fuer **alle IT-Berufe gemeinsam** (FISI, FIAE, IT-SE, etc.)

#### Struktur 2022-2023 (5 Aufgaben):
| Bereich | Punkte | Anteil |
|---------|--------|--------|
| IT 1: Unternehmen & Beschaffung | 25-30 | ~30% |
| IT 2: IT-Sicherheit | 8-24 | ~15% |
| IT 3: Netzwerktechnik | 18-24 | ~25% |
| IT 4: Datenbanken/Datenmodellierung | 15-20 | ~18% |
| IT 5: Programmierung/Softwareentwicklung | 6-10 | ~12% |

#### Struktur 2024 (4 Aufgaben - NEU!):
| Bereich | Punkte | Anteil |
|---------|--------|--------|
| IT 1: Beschaffung | 30 | 33% |
| IT 2: Programmierung | 15 | 17% |
| IT 3: Datenbank/SQL | 15 | 17% |
| IT 4: Netzwerk + Sicherheit | 30 | 33% |

**Wichtig:** Ab 2024 sind Netzwerk und IT-Sicherheit in einer Aufgabe kombiniert!

### Alte Pruefungsordnung (bis 2020)
- **GA1** (Ganzheitliche Aufgabe 1): Fachrichtungsspezifisch (FIA vs FIS)
- **GA2** (Ganzheitliche Aufgabe 2): Gemeinsam fuer alle
- **WK** (Wirtschafts- und Sozialkunde): Gemeinsam, 60 Min

## 2. Themenanalyse nach Bereich

### IT 1: Unternehmen & Beschaffung (25-30 Punkte)

#### Kernthemen:
1. **Organisationsformen**
   - Einlinienorganisation
   - Mehrlinienorganisation
   - Matrixorganisation
   - Organigramme lesen/erstellen

2. **Rechtsformen**
   - GmbH (Vorteile/Nachteile)
   - Einzelunternehmen
   - KG, GbR
   - Firmierung/Namensgebung

3. **Handelskalkulation** (SEHR WICHTIG!)
   - Bezugskalkulation (Listeneinkaufspreis -> Bezugspreis)
   - Verkaufskalkulation (Bezugspreis -> Verkaufspreis)
   - Handlungskosten, Gewinnzuschlag, Skonto, Rabatt
   - Umsatzsteuer (19%)

4. **Angebote & Beschaffung**
   - Angebotsvergleich
   - Nutzwertanalyse
   - Bestandteile eines verbindlichen Angebots
   - Bestellrhythmus- vs Bestellpunktverfahren
   - Optimale Bestellmenge

5. **Service Level Agreements (SLA)**
   - Inhalte eines SLA
   - ITIL-Begriffe: Event, Service Request, Incident

#### Typische Aufgabenformate:
- Tabellen ausfuellen (Kalkulationsschema)
- Organigramme analysieren
- Vor-/Nachteile nennen
- Rechenaufgaben mit Prozenten

---

### IT 2: IT-Sicherheit (8-24 Punkte)

#### Kernthemen:
1. **Schutzziele (CIA-Triade)**
   - Vertraulichkeit (Confidentiality)
   - Integritaet (Integrity)
   - Verfuegbarkeit (Availability)

2. **Schutzbedarfsanalyse**
   - Schutzbedarfskategorien: normal, hoch, sehr hoch
   - Maximumprinzip
   - Schutzbedarfsfeststellung durchfuehren

3. **WLAN-Sicherheit**
   - WEP, WPA2, WPA3
   - Verschluesselungsstandards
   - SSID-Broadcast
   - MAC-Filterung

4. **Bedrohungen & Gegenmassnahmen**
   - Ransomware (Handlungsempfehlungen!)
   - Social Engineering
   - Phishing
   - Malware

5. **Sicherheitsmassnahmen**
   - Firewall
   - DMZ
   - VPN
   - HTTPS vs HTTP
   - Zertifikate

#### Typische Aufgabenformate:
- Schutzbedarf in Tabelle eintragen (normal/hoch/sehr hoch)
- Sicherheitsaspekte nennen (4 Stueck)
- E-Mail beantworten (Ransomware-Beratung)
- Auswirkungen auf Schutzziele beurteilen

---

### IT 3: Netzwerktechnik (18-24 Punkte) - FISI-KERNBEREICH!

#### Kernthemen:
1. **IP-Adressierung (IPv4)**
   - Subnetting (CIDR-Notation)
   - Netzwerkadresse berechnen
   - Broadcastadresse berechnen
   - Subnetzmaske
   - Host-Bereich ermitteln

2. **IP-Adressierung (IPv6)**
   - Adresstypen: Global, Link-Local, Temporary
   - Kuerzungsregeln
   - Praefix-Notation (/64)
   - SLAAC

3. **Netzwerkdokumentation**
   - Logische vs physikalische Struktur
   - Netzwerkplaene lesen/erstellen
   - IP-Konfiguration analysieren (ipconfig-Ausgabe)

4. **DHCP**
   - DORA-Prozess (Discover, Offer, Request, Acknowledge)
   - DHCP-Parameter (IP, Subnetz, Gateway, DNS)
   - Lease-Zeit

5. **DNS**
   - Funktion erklaeren
   - Namensaufloesung

6. **WLAN-Standards**
   - IEEE 802.11 (a/b/g/n/ac/ax/Wi-Fi 6)
   - 2,4 GHz vs 5 GHz Eigenschaften
   - Reichweite, Durchsatz, Stoerungen

7. **OSI-Modell**
   - 7 Schichten
   - MAC-Adresse = Schicht 2
   - IP-Adresse = Schicht 3

8. **Netzwerkkomponenten**
   - Router, Switch, Access Point
   - PoE (Power over Ethernet)
   - Firewall

#### Typische Aufgabenformate:
- IP-Adressen vergeben und in Tabelle eintragen
- Subnetting-Berechnungen
- Netzwerkplan analysieren
- Konfigurationsausgabe interpretieren
- DHCP-Ablauf skizzieren

---

### IT 4: Datenbanken & Datenmodellierung (15-20 Punkte)

#### Kernthemen:
1. **Entity-Relationship-Modell (ERM)**
   - Entitaeten
   - Beziehungen (1:1, 1:n, m:n)
   - Kardinalitaeten
   - ERM erweitern/ergaenzen

2. **Relationenmodell**
   - Primaerschluessel
   - Fremdschluessel
   - Datentypen (INT, VARCHAR, DATE, DATETIME)
   - Relationenschema erstellen

3. **SQL**
   - SELECT mit WHERE
   - INSERT INTO
   - CREATE TABLE
   - Fehler in SQL-Statements finden

4. **Datenformate**
   - CSV
   - XML
   - JSON
   - Unterschiede erkennen

#### Typische Aufgabenformate:
- ERM um Beziehungen/Kardinalitaeten erweitern
- Relationenschema aus ERM ableiten
- SQL-Abfrage schreiben
- SQL-Fehler erklaeren
- Datenformate zuordnen

---

### IT 5: Programmierung/Softwareentwicklung (6-10 Punkte)

#### Kernthemen:
1. **Kontrollstrukturen**
   - Schleifen (for, while)
   - Verzweigungen (if/else)
   - Arrays

2. **Algorithmen**
   - Durchschnitt berechnen
   - Summe bilden
   - Fehler in Code finden

3. **Darstellungsformen**
   - Pseudocode
   - Struktogramm (Nassi-Shneiderman)
   - Programmablaufplan (PAP)
   - Programmcode

4. **Methoden/Funktionen**
   - Parameter
   - Rueckgabewerte
   - Methodenaufrufe

#### Typische Aufgabenformate:
- Methode implementieren (Pseudocode/Code)
- Struktogramm in Code umsetzen
- Fehler in Code finden und korrigieren
- Ausgabe eines Codes vorhersagen
