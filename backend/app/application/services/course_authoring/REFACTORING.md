# Course Authoring Service - Refactoring Documentation

**Date:** 2025-01-07
**Original:** `course_authoring_service.py` (975 LOC)
**Refactored into:** 8-module package (1343 LOC total, max 410 LOC per file)

## Overview

The monolithic `course_authoring_service.py` has been refactored into a modular package structure under `app/services/course_authoring/` while maintaining 100% backward compatibility.

### Quality Gates Compliance

- **G01** ✓ No duplicates - single source of truth per module
- **G02** ✓ LSX Architecture - follows Repository Pattern, no ORM
- **G04** ✓ Complete files - no code fragments
- **G05** ✓ Documentation - docstrings and type hints on all functions
- **G07** ✓ Security - parameterized queries, no secrets in code

## Module Breakdown

### 1. `exceptions.py` (8 LOC)
**Responsibility:** Exception definitions
- `CourseAuthoringError` - Base exception for all authoring errors

### 2. `validation.py` (64 LOC)
**Responsibility:** Input and operation validation
- `OperationValidator` class
  - Validates patch operations against allowed operations
  - Validates learning method types against allowed types
  - Returns validated operation list

**Key Features:**
- `VALID_OPERATIONS` - Set of 9 allowed operations (add/update/delete chapter/lesson/method + reorder + set_meta)
- `VALID_METHOD_TYPES` - Set of 10 allowed method types (calculator_tutorial, tool_tutorial, theory, quiz, etc.)

### 3. `operations.py` (157 LOC)
**Responsibility:** Draft structure patch operations
- `StructureOperations` class
  - Applies operations to draft_structure in-memory
  - Handles all 9 operation types (add/update/delete for chapters, lessons, methods)
  - Maintains operation history in meta.last_operation

**Key Methods:**
- `apply_operations()` - Main entry point, iterates operations
- `_add_chapter()`, `_update_chapter()`, `_delete_chapter()`
- `_add_lesson()`, `_update_lesson()`, `_delete_lesson()`
- `_add_method()`, `_update_method()`, `_delete_method()`

**Data Structure:**
```python
{
    'chapters': [
        {
            'id': str,
            'existing_id': str | None,  # For updates vs creates
            'title': str,
            'description': str,
            'lessons': [
                {
                    'id': str,
                    'existing_id': str | None,
                    'title': str,
                    'type': str,
                    'methods': [
                        {
                            'id': str,
                            'type': str,
                            'title': str,
                            'content': dict
                        }
                    ]
                }
            ]
        }
    ],
    'meta': {
        'version': int,
        'source': str,
        'last_operation': str | None
    },
    'activity_log': [...]  # Generated by ActivityLogGenerator
}
```

### 4. `prompts.py` (159 LOC)
**Responsibility:** AI prompt generation
- `PromptGenerator` class
  - Generates system prompts (with optional mode-specific additions)
  - Builds user prompts with course info, structure, history, files
  - Summarizes structure for compact prompt representation

**Key Methods:**
- `get_system_prompt(mode)` - Returns system prompt (base + mode-specific)
  - Modes: 'exam', 'calculator'
  - JSON format instructions for AI response
- `build_user_prompt()` - Constructs full user prompt with context
- `_summarize_structure()` - Compacts structure for display (lists chapters/lessons with method types)

### 5. `helpers.py` (189 LOC)
**Responsibility:** Data parsing and activity logging
- `DataHelpers` class
  - Parses AI responses (JSON extraction, error handling)
  - Formats chat history for prompt context
  - Extracts file context using FileContextService
- `ActivityLogGenerator` class
  - Generates human-readable activity log entries
  - Summarizes operations (e.g., "Kapitel 'Math' erstellt (3 Lektionen)")
  - Supports up to 3 individual summaries, then condenses to count

**Key Methods:**
- `parse_ai_response()` - Handles JSON in code blocks or plain JSON
- `format_history_for_prompt()` - Truncates messages to 300 chars
- `extract_file_context()` - Wraps FileContextService
- `ActivityLogGenerator.generate_entry()` - Creates log entry with timestamp, summary, operation types
- `ActivityLogGenerator._summarize_operation()` - Human-friendly operation text

### 6. `database.py` (321 LOC)
**Responsibility:** All database operations
- `DatabaseOperations` class
  - Session CRUD operations
  - Course/user permission checks
  - Structure loading from existing courses
  - Chapter/lesson/method creation/updates
  - Session updates with token/operation tracking

**Key Methods:**
- `get_course()` - Fetch course by ID
- `check_user_access()` - RBAC check (admin/teacher/creator/school/company or course owner)
- `get_course_info()` - Fetch course metadata for prompts
- `load_existing_structure()` - Recursively loads chapters→lessons from DB into draft_structure
- `update_session()` - Updates draft_structure, chat_history, file_context, token/operation counts
- `create_or_update_chapter()` - Delegates to ChapterRepository (create or update)
- `create_or_update_lesson()` - Delegates to LessonRepository (create or update)
- `create_method()` - Creates learning_method_instances with type mapping (string type → numeric LM ID)

**Type Mapping (string → LM ID):**
```python
'theory' → 0 (LM00)
'step_by_step' → 1 (LM01)
'calculator_tutorial' → 1 (LM01)
'tool_tutorial' → 9 (LM09)
'quiz' → 22 (LM22)
'flashcards' → 13 (LM13)
'exercise' → 8 (LM08)
'exam' → 19 (LM19)
'interactive' → 2 (LM02)
```

### 7. `session.py` (410 LOC) - MAIN SERVICE CLASS
**Responsibility:** Session management and main service orchestration
- `CourseAuthoringSession` class - Data container for session state
- `CourseAuthoringService` class - Main service orchestrating all modules

**CourseAuthoringSession Methods:**
- `__init__()` - Parses DB record, handles JSON deserialization
- `to_dict()` - Converts to API response format

**CourseAuthoringService Methods:**

1. `create_session(user_id, course_id, model_profile)`
   - Checks course exists, user has access
   - Loads existing structure via DatabaseOperations
   - Creates DB record in course_authoring_sessions
   - Returns session_id + draft_structure

2. `get_session(session_id, user_id)`
   - Fetches from DB
   - RBAC check
   - Returns CourseAuthoringSession.to_dict()

3. `apply_chat_message(session_id, user_id, message, mode=None, file_ids=None)`
   - Loads session
   - Extracts file context (if provided)
   - Generates prompts (system + user) via PromptGenerator
   - Calls AIAdapter.send_messages()
   - Parses response via DataHelpers.parse_ai_response()
   - Validates operations via OperationValidator.validate_operations()
   - Applies operations via StructureOperations.apply_operations()
   - Generates activity log entry via ActivityLogGenerator.generate_entry()
   - Updates session in DB via DatabaseOperations.update_session()
   - Returns: assistant_message, draft_structure, operations_applied, tokens_used

4. `finalize_session(session_id, user_id)`
   - Loads session
   - Iterates draft_structure chapters→lessons→methods
   - Creates actual DB entities via DatabaseOperations (chapters, lessons, methods)
   - Marks session as 'finalized'
   - Returns: created IDs + stats (chapter_count, lesson_count, method_count)

### 8. `__init__.py` (35 LOC)
**Responsibility:** Package initialization and public API
- Re-exports all public classes/functions
- Module docstring describing purpose
- `get_course_authoring_service()` factory function

**Exports:**
```python
CourseAuthoringService
CourseAuthoringSession
CourseAuthoringError
get_course_authoring_service()
```

## Backward Compatibility

The original file `course_authoring_service.py` is now a **bridge module** that re-exports from the package:

```python
from app.services.course_authoring import CourseAuthoringService
```

**Both import styles work:**
```python
# New style (preferred)
from app.services.course_authoring import CourseAuthoringService

# Legacy style (still works)
from app.services.course_authoring_service import CourseAuthoringService
```
